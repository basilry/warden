# WARDEN P4 영속화 및 패키징 체크리스트

## P4 목표

P4의 목표는 WARDEN을 “한 번 실행되는 demo”에서 **재시작해도 상태가 남는 로컬 PoC 패키지**로 올리는 것이다.

현재 P1의 job, approval, knowledge store는 in-memory다. P4에서는 저장소 경계를 만들고, local artifact bundle을 export/import할 수 있게 한다.

## 범위

포함:

- storage provider interface
- job persistence
- approval persistence
- knowledge unit persistence
- trace/audit retention
- report artifact retention
- export/import bundle
- offline install/runbook

제외:

- multi-tenant database
- cloud sync
- user authentication
- encrypted secret vault
- production deployment

## 선행 상태

- [x] in-memory job store 존재
- [x] in-memory approval queue 존재
- [x] in-memory knowledge store 존재
- [x] trace/audit recorder 존재
- [x] static report 계획 존재
- [x] storage provider abstraction 없음 → `src/agent/storage/*` provider contract 추가
- [x] 재시작 후 job/history 유지 없음 → JSONL provider 재생성 후 조회 검증
- [x] export/import bundle 없음 → `exportRunBundle`/`importRunBundle` 구현
- [x] offline install guide 없음 → `docs/install.md`, `docs/offline-runbook.md` 작성

## 저장소 선택 원칙

1. 첫 구현은 JSONL artifact store로 시작해도 된다.
2. SQLite는 PoC 운영성 측면에서 우선 후보다.
3. 저장소 구현은 interface 뒤로 숨긴다.
4. 저장소 선택이 agent contract를 바꾸면 안 된다.
5. trace/audit은 append-only 성격을 유지한다.

## 개발 순서

1. P4.0 Storage provider contract
2. P4.1 JSONL artifact store
3. P4.2 SQLite store
4. P4.3 Export/import run bundle
5. P4.4 Packaging and offline runbook

## P4.0 Storage Provider Contract

### 목표

현재 in-memory store들을 공통 storage interface 뒤로 이동한다.

### 생성 파일

| 파일 | 목적 |
|---|---|
| `src/agent/storage/types.ts` | storage provider 공통 타입 |
| `src/agent/storage/provider.ts` | provider factory |
| `src/agent/storage/memory-store.ts` | 기존 in-memory 구현 이동 |
| `src/agent/config.ts` | runtime config |

### 수정 파일

| 파일 | 수정 내용 |
|---|---|
| `src/agent/jobs.ts` | storage provider 사용 |
| `src/agent/approval.ts` | storage provider 사용 |
| `src/agent/knowledge/store.ts` | storage provider 사용 |
| `src/agent/p1-runner.ts` | config 기반 store 생성 |
| `src/agent/run-store.ts` | artifact writer와 storage 연계 |

### 핵심 타입

```ts
type StorageProviderKind = "memory" | "jsonl" | "sqlite";

type StorageProvider = {
  kind: StorageProviderKind;
  jobs: JobRepository;
  approvals: ApprovalRepository;
  knowledge: KnowledgeRepository;
  traces: TraceRepository;
  artifacts: ArtifactRepository;
};
```

### 핵심 함수

- `loadWardenConfig(env: NodeJS.ProcessEnv): WardenConfig`
- `createStorageProvider(config: WardenConfig): StorageProvider`
- `createMemoryStorageProvider(): StorageProvider`
- `saveJob(job: JobRecord): Promise<void>`
- `loadJob(jobId: string): Promise<JobRecord | undefined>`
- `listJobs(filter?: JobFilter): Promise<JobRecord[]>`
- `saveApproval(item: ApprovalItem): Promise<void>`
- `saveKnowledgeUnit(unit: KnowledgeUnit): Promise<void>`
- `appendTraceEvent(event: TraceEvent): Promise<void>`

## P4.2 SQLite Store

### 목표

PoC 운영을 위해 query 가능한 local database를 붙인다. 단, SQLite dependency가 과하면 JSONL store를 먼저 완료하고 P4.2를 후속으로 둔다.

### 생성 파일

| 파일 | 목적 |
|---|---|
| `src/agent/storage/sqlite-store.ts` | SQLite provider |
| `src/agent/storage/schema.ts` | schema definition |
| `src/agent/storage/migrations.ts` | migration runner |
| `src/agent/storage/sql.ts` | query helper |

### 테이블 초안

| 테이블 | 주요 컬럼 |
|---|---|
| `jobs` | `id`, `status`, `request`, `created_at`, `updated_at`, `payload_json` |
| `approvals` | `id`, `job_id`, `status`, `risk`, `tool_name`, `payload_json`, `created_at` |
| `knowledge_units` | `id`, `source_uri`, `source_type`, `hash`, `payload_json`, `created_at` |
| `trace_events` | `id`, `run_id`, `task_id`, `phase`, `actor`, `summary`, `payload_json`, `ts` |
| `artifacts` | `id`, `run_id`, `kind`, `path`, `hash`, `created_at` |

### 핵심 함수

- `createSqliteStorageProvider(dbPath: string): StorageProvider`
- `runMigrations(db: SqliteDb): void`
- `insertJob(db: SqliteDb, job: JobRecord): void`
- `upsertApproval(db: SqliteDb, item: ApprovalItem): void`
- `insertKnowledgeUnit(db: SqliteDb, unit: KnowledgeUnit): void`
- `insertTraceEvent(db: SqliteDb, event: TraceEvent): void`
- `listTraceEvents(runId: string): TraceEvent[]`

## P4.3 Export/Import Run Bundle

### 생성 파일

| 파일 | 목적 |
|---|---|
| `src/agent/storage/bundle.ts` | run bundle export/import |
| `demo/run-warden-export-demo.ts` | export demo |

### Bundle 구성

| 파일 | 내용 |
|---|---|
| `manifest.json` | run id, generatedAt, version, hashes |
| `job.json` | job record |
| `approvals.json` | approval queue snapshot |
| `knowledge.json` | knowledge units |
| `trace.jsonl` | trace events |
| `brief.md` | audit brief |
| `report.html` | P3 HTML report |

### 핵심 함수

- `exportRunBundle(runId: string, storage: StorageProvider, outputDir: string): Promise<RunBundle>`

- `hashBundleFile(path: string): string`
