# WARDEN 전체 마일스톤 구현 상태 점검

## 결론

현재 `/Users/basilry/Projects/02021_warden_agents`는 **P1 offline control-plane MVP**까지 구현됐다.

즉, WARDEN의 핵심 명제인 “에이전트 팀 실행 → 정책 게이트 → 결정적 분석 → 독립 검증 → 감사 브리프 → 회귀검증”은 로컬 deterministic demo로 닫혔다. P1에서는 여기에 모델 어댑터 경계, capability 기반 MCP manager, approval queue, job history, knowledge ingestion/store까지 붙었다.

다만 아직 **제품 운영 단계**는 아니다. live LLM, 실제 stdio MCP 연결, SourceVet reviewer, 승인 UI, 웹 콘솔, 영속 저장소, 실제 문서 파서, 패키징/배포, 보안 hardening은 다음 페이즈로 남아 있다.

현재 상태를 한 줄로 정리하면:

> WARDEN은 “통제형 멀티에이전트 하네스의 검증 가능한 로컬 MVP”까지 왔고, 다음 단계는 “실제 연동·UI·영속화·도메인 확장”이다.

## 구현 수준 판정

### 기술 성숙도

현재 수준: **Prototype+ / Local MVP**

근거:

- 실행 가능한 CLI demo가 있다.
- policy, trace, verifier, regression이 실제로 동작한다.
- 외부 네트워크 없이 재현 가능하다.
- P1 control-plane 컴포넌트가 연결되어 있다.
- 하지만 모든 저장소가 in-memory이고, live MCP/LLM/UI가 없다.

### 제품 성숙도

현재 수준: **심사용 기술 데모**

근거:

- 3분 데모에서 “왜 방산에 필요한지” 설명 가능.
- job history, approval pending, audit brief가 출력된다.
- 다만 비개발자가 직접 쓰기 위한 UI는 없다.

### 보안 성숙도

현재 수준: **보안 원칙 구현 시작**

근거:

- no silent external call 원칙이 regression으로 잠겼다.
- policy 없는 WRITE가 실패한다.
- 모델 output은 실행권이 아니라 proposal로 취급한다.
- 다만 실제 권한관리, 사용자 인증, 로그 무결성, 저장소 암호화는 없다.

## 남은 전체 작업량 추정

| 구간 | 설명 | 남은 작업량 |

## 2026-06-15 실사용 CLI 테스트 반영 업데이트

### 사용자 테스트

입력:

```text
대한민국 및 동북아 공급망에 대해 알려줘
```

관찰:

- Codex CLI model proposal은 2회 호출되었고 각 호출은 약 24초 걸렸다.
- 런타임은 1회차에 `run_warden_team`, 2회차에 `external_osint_fetch`를 실행 계획으로 선택했다.
- `external_osint_fetch`는 EXTERNAL risk라 승인 대기 상태에서 멈췄다.
- 최종 출력은 실행 상태, ACH 생존 가설, 승인 대기열 중심이었다.
- 사용자가 기대한 실제 설명형 답변은 출력되지 않았다.

### 재판정

현재 WARDEN은 다음 범위까지 구현됐다.

- agent team runner
- policy gate
- ACH deterministic authority
- SourceVet reviewer
- approval queue
- storage/persistence
- runtime server
- `warden` CLI entrypoint
- Codex OAuth 기반 live proposal call
- regression pack
- README와 GitHub push

그러나 다음은 아직 미구현이다.

- 사용자 질문에 대한 final answer composer
- 모델 proposal을 안전하게 tool plan 후보로 반영하는 planner wiring
- 승인 후 같은 run을 재개하는 approve/resume UX
- 승인된 외부 근거 수집 결과를 SourceVet과 답변에 반영하는 루프
- 한국/동북아 공급망 질문에 맞는 domain grounding/retrieval
- 긴 model latency 동안의 진행 표시 UX

따라서 현재 성숙도는 다음으로 수정한다.

| 영역 | 기존 판정 | 수정 판정 |
|---|---|---|
| 통제형 하네스 | Local MVP | 유지 |
| 제출/심사용 기술 데모 | 가능 | 유지 |
| 실제 사용자 답변 제품 | 부분 가능 | 미완성 |
| live LLM 활용 | proposal call 가능 | answer synthesis 미연결 |
| live MCP/OSINT | 승인 대기까지 가능 | 승인 후 resume/fetch 미완성 |

핵심 결론:

> P0~P6는 “안전하게 에이전트 루프를 돌리는 기반”이고, 실사용자가 기대하는 “질문에 대한 분석 답변”은 P7부터 새로 닫아야 한다.
