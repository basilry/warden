# WARDEN 안보 지정학 공급망 미래예측 특화 개발계획

## 0. 현재 상태 요약

현재 WARDEN은 controlled multi-agent runtime의 골격은 확보했다.

- CLI 실행: `warden`, `warden run "<objective>"`, `warden server`
- Codex OAuth 기반 모델 proposal 경로
- policy gate, approval queue, y/n 승인 UX
- SourceVet, ACH, verifier, deterministic answer composer
- OSINT MCP 도구: `search_news`, `scrape_news`, `discover_news`
- live OSINT opt-in: `WARDEN_OSINT_LIVE_OPT_IN=true`
- 회귀 테스트: `npm test` 통과
- 최신 커밋: `638c280 Add OSINT discovery MCP and approval prompts`
- 2026-06-18 작업분: P19/P20 기반 구현 완료. `InvestigationPlan` 기반 질문별 가설/검색계획 생성, dynamic CaseFrame, dynamic evidence-to-ACH mapping, CLI `[분석계획]` 로그, P19/P20 regression이 추가되었다.
- 2026-06-18 병렬 개발 업데이트: P21 RAG MCP, P22 OSINT source registry, P23 claim graph/evidence ledger, P24 forecast engine, P25 security report composer, P26 domain ontology/scenario library, P27 MCP boundary validator, P28 golden fixture foundation을 병렬 구현했다. `npm test` 전체 회귀 통과. 현재 작업분은 아직 커밋 전이다.

하지만 사용자가 원하는 최종 형태와 비교하면 아직 보고서/RAG/예측 루프가 부족하다.

- `case-framer.ts`는 `investigationPlan`이 있으면 dynamic CaseFrame을 사용하고, 없을 때만 legacy fixture frame으로 fallback한다.
- `evidence-curator.ts`는 `investigationPlan`과 `KnowledgeUnit[]`가 있으면 dynamic EvidenceBundle을 생성한다.
- `src/runtime/planner.ts`는 여전히 도구 선택 proposal을 담당하고, 별도 `src/runtime/investigation-planner.ts`가 질문별 조사계획을 담당한다.
- live OSINT와 dynamic ACH에 더해 로컬 RAG, ontology expansion, claim graph, forecast scoring, security report composer가 런타임 산출물에 연결됐다. 남은 큰 공정은 실제 웹 discovery/scrape 품질, RAG corpus ingest 확장, MCP 경계 완전 통일, 평가셋 확대다.

따라서 다음 개발 목표는 "통제형 런타임"에서 "안보/지정학/공급망 특화 분석 에이전트"로 승격하는 것이다.

## 1. 제품 목표

WARDEN은 자연어 질문을 받으면 다음 흐름으로 답해야 한다.

1. LLM이 질문을 해석하고 조사 범위, 시간 지평, 지역, 행위자, 핵심 불확실성을 잡는다.
2. LLM이 가설과 분석계획, 검색계획, RAG 계획, 필요 지표를 구조화한다.
3. MCP들이 웹 discovery/scrape, 기존 자료 RAG, ACH, SourceVet, forecast scoring을 실행한다.
4. 근거가 있으면 근거 기반으로 판단하고, 근거가 부족하면 부족하다고 말한 뒤 추정값과 불확실성을 분리한다.
5. 최종 보고서는 "사실, 추론, 예측, 불확실성, 확인해야 할 지표"를 구분해 생성한다.

목표 도메인:

- 안보: 군사 동향, 분쟁 위험, 제재, 군수 조달, 방산 공급망
- 지정학: 국가 행위자, 외교 신호, 동맹, 역내 긴장, 정책 변화
- 공급망: 반도체, 배터리, 희토류, 해운, 에너지, 수출통제, 핵심 부품
- 미래예측: 확률적 전망, 시나리오, 지표 기반 조기경보, 업데이트 트리거

3. ACH/forecast deterministic scoring

## 15. 성공 상태 정의

WARDEN이 사용자가 원하는 에이전트가 됐다고 볼 수 있는 기준:

- 자연어 질문만으로 조사계획을 세운다.
- 하드코딩된 가설 없이 질문별 분석틀을 만든다.
- 웹, RAG, 공식자료, 뉴스자료를 MCP로 수집한다.
- 증거 부족 시 부족하다고 말한다.
- 안보/지정학/공급망의 행위자, 지표, 시나리오를 이해한다.
- 예측은 기간, 확률, 근거, 반증지표, 업데이트 조건을 포함한다.
- 최종 보고서가 근거와 추론을 분리한다.
- 모든 외부 호출은 approval/policy trace에 남는다.
- 회귀 테스트와 eval로 품질이 유지된다.
