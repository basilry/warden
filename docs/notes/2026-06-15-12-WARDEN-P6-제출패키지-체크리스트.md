# WARDEN P6 제출 패키지 체크리스트

## P6 목표

P6의 목표는 WARDEN을 방산 스타트업 챌린지 또는 수요기업 PoC 매칭에서 설명 가능한 제출 패키지로 묶는 것이다.

제품 메시지는 다음 한 문장으로 고정한다.

> WARDEN은 방산 조직이 LLM을 직접 신뢰하지 않고도 활용할 수 있도록, 모든 에이전트 실행을 정책, 승인, 결정적 분석, 독립 검증, 감사 로그로 통제하는 offline-first 멀티에이전트 하네스다.

## 범위

포함:

- one-pager
- 3분 demo script
- demo video shot list
- architecture diagram
- security/OPSEC explanation
- install/run guide
- evaluation checklist
- 수요기업 과제 매칭 키워드
- FAQ

제외:

- 과장된 live autonomy 주장
- 실제 군 내부망 적용 완료 주장
- 인증되지 않은 보안 인증 보유 주장
- 고객 레퍼런스가 없는 상태의 실적 주장

## P6.0 제출 Narrative 고정

### 목표

심사위원, 수요기업, 기술검토자가 같은 메시지를 받도록 표현을 고정한다.

### 생성 파일

| `docs/submission/use-cases.md` | 수요기업 use case |

### 핵심 메시지

- 방산 분석에서 LLM의 약점은 답변 품질만이 아니라 통제, 승인, 감사, 재현성 부족이다.
- WARDEN은 LLM을 판단자가 아니라 specialist proposer와 tool orchestrator로 제한한다.
- ACH와 SourceVet 같은 결정적 규율 모듈이 분석 절차와 출처 검증을 강제한다.
- Policy Reviewer와 HITL approval이 외부 호출과 위험 작업을 막는다.
- Trace와 regression이 실패를 재현 가능한 자산으로 바꾼다.

| `docs/submission/demo-checklist.md` | 녹화 전 점검 |

### 3분 흐름

| 시간 | 화면 | 메시지 |
|---:|---|---|
| 0:00-0:20 | 문제 정의 | 방산 분석에서 AI 답변보다 통제와 감사가 중요 |
| 0:20-0:50 | WARDEN team run | Supervisor가 specialist team을 지휘 |
| 0:50-1:20 | ACH result | 결론은 LLM이 아니라 결정적 ACH가 계산 |
| 1:20-1:45 | SourceVet/Policy | 출처와 tool call은 별도 reviewer가 검증 |
| 1:45-2:15 | Approval/Trace | 외부 호출은 pending, 모든 handoff가 trace |
| 2:15-2:45 | HTML report | 비개발자도 audit report로 확인 |
| 2:45-3:00 | PoC 제안 | offline-first로 수요기업 환경에서 PoC 가능 |
