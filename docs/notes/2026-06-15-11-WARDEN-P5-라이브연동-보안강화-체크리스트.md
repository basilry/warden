# WARDEN P5 라이브 연동 및 보안 강화 체크리스트

## P5 목표

P5의 목표는 WARDEN을 실제 LLM, 실제 stdio MCP, 실제 문서 ingestion과 연결하되, P0부터 지킨 원칙을 깨지 않는 것이다.

핵심 원칙:

- live mode는 명시적 opt-in이다.
- model output은 실행 명령이 아니라 proposal이다.
- 외부 호출은 policy review와 approval 없이는 실행되지 않는다.
- secret, prompt, document payload는 로그에 그대로 남기지 않는다.
- deterministic authority 값은 LLM이 덮어쓸 수 없다.

## 범위

포함:

- live model adapter
- local model adapter 후보
- stdio MCP 실제 invoke
- PDF/HTML/text ingestion parser
- egress policy
- redaction
- output authority validator
- no-egress regression

제외:

- 인터넷 자동 수집 agent
- 무승인 외부 action
- production secret manager
- SOC/SIEM integration
- 인증/권한관리 전체 구현

## P5.0 Live Model Adapter Opt-in

### 목표

P1의 model adapter contract를 유지하면서 live model을 선택적으로 연결한다. 기본값은 계속 mock/offline이다.

### 생성 파일

| 파일 | 목적 |
|---|---|
| `src/agent/models/live-openai-compatible.ts` | explicit opt-in live adapter |
| `src/agent/models/local-model.ts` | local model adapter 후보 |
| `src/agent/models/model-config.ts` | model config parser |
| `src/agent/security/model-output-validator.ts` | model output authority 검증 |

### 수정 파일

| 파일 | 수정 내용 |
|---|---|
| `src/agent/model-adapter.ts` | live mode contract 명시 |
| `src/agent/models/openai-compatible.ts` | dry-run과 live adapter 분리 |
| `src/agent/p1-runner.ts` | model config 주입 |
| `src/agent/policy.ts` | model-suggested tool call risk 연결 |

### 핵심 함수

- `loadModelConfig(env: NodeJS.ProcessEnv): ModelConfig`
- `createModelAdapter(config: ModelConfig): ModelAdapter`
- `createLiveOpenAiCompatibleAdapter(config: LiveModelConfig): ModelAdapter`
- `createLocalModelAdapter(config: LocalModelConfig): ModelAdapter`
- `assertLiveModelOptIn(config: ModelConfig): void`
- `validateModelOutputAgainstAuthority(output: ModelOutput, authorities: AuthoritySnapshot): ValidationReport`
- `rejectToolExecutionFromRawModelOutput(output: ModelOutput): PolicyDecision`

### 구현 로직

1. env/config에 `WARDEN_MODEL_MODE=mock|dry_run|live|local`을 둔다.
2. `live`는 `WARDEN_LIVE_MODEL_OPT_IN=true` 없이는 생성 실패한다.
3. API key는 runtime에만 쓰고 trace payload에 남기지 않는다.
4. model output이 tool call을 제안해도 즉시 실행하지 않는다.
5. model output은 `ToolCallPlan`으로 변환된 뒤 policy reviewer를 통과한다.
6. ACH ranking, SourceVet risk, policy decision 같은 authority 값은 model text가 수정할 수 없다.

### 체크리스트

- [x] model config parser 구현 → `src/agent/config.ts`에 model provider config 추가
- [x] live opt-in guard 구현 → `WARDEN_CODEX_DRY_RUN=0` 명시 필요
- [x] API key presence validation 구현
- [x] dry-run adapter와 live adapter 분리 → Codex CLI adapter에서 dry-run/live branch 분리
- [x] local model adapter 후보 구현
- [x] raw model tool execution 금지 → P1은 model output을 proposal로만 저장
- [x] model output authority validator 구현
- [x] model output redaction 적용
- [x] live disabled regression 추가

## P5.1 Real stdio MCP Invoke

### 목표

P2에서 만든 MCP boundary에 실제 stdio server discovery/invoke를 붙인다.

### 생성/수정 파일

| 파일 | 목적 |
|---|---|
| `src/agent/mcp/stdio-client.ts` | 실제 stdio lifecycle |
| `src/agent/mcp/protocol.ts` | MCP message encode/decode helper |
| `src/agent/mcp/sandbox.ts` | command allowlist guard |
| `fixtures/mcp/local-ach-server.json` | 테스트용 MCP config |

### 핵심 함수

- `startMcpServer(config: McpServerConfig): Promise<McpServerProcess>`
- `stopMcpServer(server: McpServerProcess): Promise<void>`
- `sendMcpRequest(server: McpServerProcess, request: McpRequest): Promise<McpResponse>`
- `discoverRemoteTools(server: McpClient): Promise<RemoteToolDescriptor[]>`
- `invokeRemoteTool(server: McpClient, toolName: string, input: unknown): Promise<ToolResult>`
- `assertMcpCommandAllowed(config: McpServerConfig, allowlist: CommandAllowlist): void`

### 구현 로직

1. stdio server command는 config allowlist를 통과해야 한다.
2. server process start/stop은 trace에 남긴다.
3. tool discovery 결과는 capability allowlist와 risk classification을 통과해야 한다.
4. remote invoke는 policy reviewer와 approval queue를 통과해야 한다.
5. server error, timeout, malformed response는 fail closed로 처리한다.
6. 모든 remote result는 schema validation 뒤 specialist agent에 전달된다.

| 파일 | 목적 |

### 핵심 함수

- `ingestDocument(path: string, options: IngestOptions): Promise<KnowledgeUnit[]>`
- `parseTextDocument(buffer: Buffer, metadata: DocumentMetadata): ParsedDocument`
- `parseHtmlDocument(buffer: Buffer, metadata: DocumentMetadata): ParsedDocument`
- `parsePdfDocument(buffer: Buffer, metadata: DocumentMetadata): ParsedDocument`
- `hashDocument(buffer: Buffer): string`
- `buildDocumentProvenance(path: string, hash: string, parserVersion: string): Provenance`
- `extractClaims(parsed: ParsedDocument): Claim[]`

### 구현 로직

1. parser는 raw text를 직접 ACH/SourceVet에 넘기지 않는다.
2. parser output은 `KnowledgeUnit`과 `Claim`으로 정규화한다.
3. 모든 unit은 document hash, parser version, source path, extractedAt을 가진다.
4. PDF parser가 실패하면 partial ingestion을 숨기지 않고 warning으로 남긴다.
5. ingestion 결과는 SourceVet review 대상이 된다.
