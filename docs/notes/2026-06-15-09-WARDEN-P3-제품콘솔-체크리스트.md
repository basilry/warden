# WARDEN P3 제품 콘솔 및 감사 리포트 체크리스트

## P3 목표

P3의 목표는 CLI demo를 **비개발자가 이해할 수 있는 제품 화면**으로 바꾸는 것이다.

첫 단계는 웹앱보다 static HTML audit report가 낫다. 이유는 간단하다. 현재 WARDEN은 offline/local 재현성이 핵심이고, static report는 서버, 인증, 배포 없이도 심사와 녹화에 바로 쓸 수 있다.

## 범위

포함:

- HTML audit report export
- job summary panel
- approval queue panel
- policy decision panel
- SourceVet risk panel
- ACH matrix and survivor panel
- trace timeline panel
- regression result panel

제외:

- 다중 사용자 인증
- 실시간 협업
- 원격 배포
- live MCP/LLM 실행 제어
- 장기 운영 dashboard

## P3.0 Report Data Contract

### 목표

P0/P1/P2 산출물을 화면 전용 구조로 정규화한다. 렌더러가 agent 내부 타입에 과도하게 의존하지 않게 한다.

### 생성 파일

| 파일 | 목적 |
|---|---|
| `src/agent/report/types.ts` | report view model 타입 |
| `src/agent/report/build-report.ts` | run result에서 view model 생성 |
| `src/agent/report/renderers.ts` | markdown/html 공통 section renderer |

### 핵심 타입

```ts
type WardenReport = {
  reportId: string;
  runId: string;
  generatedAt: string;
  title: string;
  status: "pass" | "warn" | "fail" | "blocked";
  casePanel: CasePanel;
  achPanel: AchPanel;
  sourceVetPanel?: SourceVetPanel;
  policyPanel: PolicyPanel;
  approvalPanel: ApprovalPanel;
  tracePanel: TracePanel;
  regressionPanel?: RegressionPanel;
};
```

### 핵심 함수

- `buildWardenReport(input: WardenReportInput): WardenReport`
- `buildCasePanel(result: TeamRunResult): CasePanel`
- `buildAchPanel(result: TeamRunResult): AchPanel`
- `buildSourceVetPanel(review: SourceReview): SourceVetPanel`
- `buildPolicyPanel(events: TraceEvent[]): PolicyPanel`
- `buildApprovalPanel(queue: ApprovalQueueSnapshot): ApprovalPanel`
- `buildTracePanel(events: TraceEvent[]): TracePanel`

### 체크리스트

- [x] report view model 타입 정의
- [x] team run result에서 case panel 생성
- [x] ACH result에서 matrix/survivor panel 생성
- [x] SourceVet result에서 risk panel 생성
- [x] trace events에서 policy panel 생성
- [x] approval queue에서 pending panel 생성
- [x] trace timeline panel 생성
- [x] regression result panel optional 처리

## P3.1 Static HTML Audit Report

### 목표

`npm run demo:warden:report` 실행 시 `reports/<runId>/index.html`을 생성한다.

### 생성 파일

| 파일 | 목적 |
|---|---|
| `src/agent/report/html-report.ts` | HTML report renderer |
| `src/agent/report/assets.ts` | inline CSS와 report asset helper |
| `demo/run-warden-report-demo.ts` | report 생성 demo |
| `reports/.gitkeep` | report output 위치 표시 |

### 수정 파일

| 파일 | 수정 내용 |
|---|---|
| `.gitignore` | `reports/*` ignore, `.gitkeep` 예외 |
| `package.json` | `demo:warden:report` script 추가 |
| `README.md` | report demo 명령 추가 |

### 핵심 함수

- `renderHtmlReport(report: WardenReport): string`
- `renderReportShell(report: WardenReport, body: string): string`
- `renderCasePanel(panel: CasePanel): string`
- `renderAchPanel(panel: AchPanel): string`
- `renderSourceVetPanel(panel: SourceVetPanel): string`
- `renderPolicyPanel(panel: PolicyPanel): string`
- `renderApprovalPanel(panel: ApprovalPanel): string`
- `renderTraceTimelinePanel(panel: TracePanel): string`
- `writeReportArtifacts(outputDir: string, report: WardenReport): ReportArtifact`

### UI 원칙

- 첫 화면에서 case status, survivor hypothesis, approval pending count, source risk count가 보인다.
- 장식보다 감사 가능성이 우선이다.
- 각 panel은 독립적으로 스캔 가능해야 한다.
- 색은 status 전달에만 사용한다.
- trace timeline은 전체 흐름을 시간 순서대로 보여준다.
- report는 서버 없이 파일로 열 수 있어야 한다.
