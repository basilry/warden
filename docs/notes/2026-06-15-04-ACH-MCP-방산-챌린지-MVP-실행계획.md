# ACH MCP 방산 챌린지 MVP 실행계획

## 목표

후속 방산 스타트업 챌린지 스타트업 모집/매칭 단계에 맞춰, 현재 `ach-mcp`를 **통제형 AI 분석 보조 MVP**로 확장한다.

MVP의 핵심은 모델 성능이 아니라 다음 네 가지를 보여주는 것이다.

1. LLM이 판단하지 않는다.
2. 판단은 MCP의 결정적 규율 엔진이 한다.
3. 모든 툴 호출은 정책·승인·감사를 거친다.
4. 방산 수요기업이 이해 가능한 업무 시나리오로 작동한다.

## MVP 범위

### 포함

- ACH 분석 케이스 데모
- SourceVet 출처검증 데모
- WARDEN 통제형 에이전트 루프 P0
- 정책 엔진: READ/WRITE/DESTRUCTIVE/EXTERNAL
- HITL 승인 mock
- agent-level audit log
- 심사용 UI 또는 녹화 가능한 로컬 웹 콘솔
- 합성 방산 공급망/공개출처 리스크 시나리오

### 제외

- 실작전/실데이터 연결
- 외부 OSINT 실시간 크롤링
- 모델 파인튜닝
- 멀티테넌트 권한관리
- 폐쇄망 실제 인증/인가 통합
- 방산망 실배포

## 1차 시나리오

### 제목

방산 공급망 이상징후 분석: 핵심 부품 수입 급감의 원인 판단

### 흐름

1. 사용자가 자연어로 상황을 입력한다.
2. 에이전트가 경쟁가설을 제안한다.
3. MCP가 가설 수 부족, 무출처 증거, 미평가 칸을 거부한다.
4. 증거별 신뢰도와 변별력을 계산한다.
5. 최소모순 가설을 산출한다.
6. 생존 가설을 가르는 RFI를 만든다.
7. 새 hit 주입 후 판단이 바뀔 때만 알림을 보낸다.
8. audit timeline으로 전 과정을 재현한다.

## 2차 시나리오

### 제목

협력사/출처 보고 신뢰도 검증: 기대와 일치하는 보고의 날조 위험 탐지

### 흐름

1. 두 출처가 서로 다른 보고 이력을 갖는다.
2. 운영관 직감은 기대에 맞는 출처로 기운다.
3. SourceVet이 접근범위, 검증불가율, echo, circular lineage를 계산한다.
4. 독립검증 없는 신뢰도 상향을 거부한다.
5. 검증요구를 생성한다.

## 성공 기준

### 기술

- `npm run build` 통과
- ACH/SourceVet 자동 데모 통과
- WARDEN 데모 통과
- 도구 호출이 정책 엔진을 우회하지 않음
- audit log로 세션 재현 가능

### 제품

- 비개발자가 3분 내에 “왜 방산에 필요한지” 이해한다.
- 심사위원이 외부 API 없이 데모를 실행할 수 있다.
- 수요기업 과제 하나 이상에 맞춰 설명을 바꿀 수 있다.

### 보안

- 실데이터 없음
- 외부 전송 없음
- 합성 시나리오 명시
- 모델 판단권 없음
- 사람 승인 구조 명시

## 제출용 한 문장

> WARDEN은 방산 조직이 LLM을 직접 신뢰하지 않고도 활용할 수 있도록, MCP 기반 도구 호출을 정책·승인·감사로 통제하고, 분석 판단은 ACH/SourceVet 같은 결정적 규율 엔진에 맡기는 폐쇄망 친화 AI 분석 보조 MVP입니다.
